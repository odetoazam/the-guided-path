## Purpose

This document is read by the hub-article-pipeline skill at the start of every session. It answers four questions the article backlog (a checklist) cannot:

1. **What angles have already been taken?** So the pipeline doesn't duplicate.
2. **What cross-entity threads are implicit in existing articles but not yet written?** So editorial decisions are driven by what the graph actually reveals, not memory.
3. **Where are the contextual link opportunities?** Specific moments in existing articles where a link to another article would illuminate a shared pattern — not keyword matching, but semantic bridging.
4. **What is the highest-value untouched territory?** Prioritized by rarity of angle, not just entity occurrence count.
